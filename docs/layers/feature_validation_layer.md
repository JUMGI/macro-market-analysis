# Feature Validation Layer

## Overview

The **Feature Validation Layer** is responsible for evaluating the **quality and usefulness of features** generated by the Systemic Layer.

It does **not modify data**.  
It acts as an **observability and diagnostics layer** over features.

---

## Purpose

This layer answers:

- Which features are stable?
- Which features contain noise or missing data?
- Which features are redundant?
- Which features are suitable for downstream tasks (regime detection, alpha models)?

---

## Position in the Architecture

```
Systemic Layer
↓
Feature Validation Layer
↓
Research Layer
```

---

## Key Principles

### 1. Separation of Concerns

- Systemic Layer → produces data
- Feature Validation → evaluates data
- Research Layer → decides what to use

---

### 2. Config-Driven

Validation behavior is fully controlled via configuration files.

Example:

```json
{
    "metrics": {
        "stability": {
            "enabled": true,
            "params": { "window": 63 }
        }
    }
}
```
---

### 3. Deterministic & Reproducible

Each validation run is uniquely identified by:

dataset_hash + config → fv_hash

This ensures:

- reproducibility
- comparability
- caching

---

### 4. Non-Intrusive

The layer:

- does NOT change features
- does NOT filter features automatically
- does NOT make decisions

It only produces metadata

---

## Inputs

- Systemic feature dataset (`DataFrame`)
- Systemic metadata (`dataset_hash`)
- Validation config

---

## Outputs

A FeatureMetadata artifact containing:

- dataset_id
- dataset_hash
- fv_hash
- metrics per feature
- config used

---

## Example Output
{
    "dataset_id": "baseline_v2",
    "dataset_hash": "...",
    "fv_hash": "...",
    "metrics": {
        "mean_MOM_5": {
            "stability": 0.82,
            "autocorr": 0.65,
            "missing": 0.0,
            "redundancy": 0.34
        }
    }
}

---

## Metrics

Typical metrics include:

- Stability (rolling variance consistency)
- Autocorrelation (persistence)
- Missing values
- Redundancy (cross-feature correlation)

## Extensibility

New metrics can be added by:

- Implementing a function
- Registering it in the metric registry
- Enabling it via config

No changes to the profile are required.

---
## What This Layer Is NOT

- Not a feature engineering layer
- Not a modeling layer
- Not a feature selection engine

---
## Role in Research

This layer provides:

- feature diagnostics
- ranking signals
- filtering capabilities

Used later in:

- regime detection
- alpha modeling
- allocation strategies

---

## Summary

The Feature Validation Layer transforms raw features into research-ready metadata, enabling:

- systematic feature evaluation
- reproducible experiments
- scalable research workflows
