# Panel Preparer Architecture

## Overview

The **Panel Preparer** is the final stage before systemic feature generation.

It transforms a raw multi-asset panel into a **clean, model-ready dataset**, ensuring consistency, alignment, and data quality.

It is a deterministic, config-driven step that guarantees reproducibility and removes data artifacts before systemic aggregation.

---

## Position in Architecture


Asset Features (MOM, VOL, RET, etc.)
        ↓
Panel Builder (estructura + raw panel)
        ↓
Panel Preparer (cleaning + alignment + NaNs)
        ↓
Systemic Pipeline (cross-asset aggregation)
        ↓
Regime Layer
        ↓
Strategy Layer


---

## Objective

The Panel Preparer ensures that:

- All assets are properly aligned in time
- Missing data is handled consistently
- The dataset is free of structural artifacts
- Feature distributions are stable enough for aggregation
- Output is deterministic and reproducible

---

## Input

### Raw Panel

A multi-asset DataFrame:

- index: datetime
- columns: MultiIndex [asset, feature]

Generated by the **Panel Builder**.

---

### Configuration

Controlled via:

```python
CONFIG["panel"]["nan_handling"]
CONFIG["panel"]["alignment"]
```

## Processing Stages
### 1. Alignment Strategy

Defines how assets are synchronized in time.

Options:
- `union`
- `intersection`
Behavior:
- `union`:
- keeps full calendar
- introduces NaNs where data is missing
- `intersection`:
- keeps only common dates
- reduces missing values

### 2. Warmup Handling

Removes unstable initialization period.

Options:
- `cut` → trims until first valid observation
- `keep` → preserves full history
Based on:
- `used_features`
- or `all`

### 3. Calendar Alignment

Handles missing values across time.

Options:
- ffill → forward fill (default in financial time series)
- bfill → backward fill
- drop → remove rows with NaNs
- none → no operation

### 4. Final Cleaning

Ensures dataset is model-ready.

Options:
- dropna → remove any remaining NaNs
- keep → allow residual NaNs (rare use case)


## Output
*Clean Panel*
`panel_clean`

Properties:

- fully aligned
- cleaned according to config
- deterministic transformation

---

## Design Principles
### 1. Separation of concerns

| Layer     | Responsibility   |
|----------|------------------|
| Builder   | Structure        |
| Preparer  | Data quality     |
| Systemic  | Market structure  |

---

### 2. Config-driven logic

All transformations are defined in:
```python
CONFIG["panel"]
```
No hardcoded behavior.

### 3. Deterministic output

Given the same:

- input panel
- config

The output is identical.

### 4. No feature engineering

The Panel Preparer does NOT:

- create features
- transform signals
- aggregate across assets

It only prepares data.

## Output Contract

The Panel Preparer guarantees:

- consistent index across assets
- no structural misalignment
- controlled missing data strategy
- stable input for systemic aggregation


## Validation Layer

Optional checks:

- NaN ratio after cleaning
- continuity of index
- extreme value detection
- asset coverage consistency

## Common Failures Prevented
- misaligned trading calendars
- partial asset overlap
- inconsistent warmup periods
- hidden NaN propagation into systemic layer

## Role in Systemic Stability

The quality of systemic features depends directly on:

- Panel Preparer correctness

Errors here propagate non-linearly into:

- dispersion metrics
- correlation structure
- breadth signals

## Summary

The Panel Preparer acts as the data sanitation boundary between raw asset data and systemic market modeling.

It ensures that all downstream layers operate on a consistent, stable, and reproducible market representation.
